# 场景配置

场景配置定义仿真的初始状态。本文档描述场景配置的概念和结构。

## 什么是场景

场景是一次仿真的完整配置，包括：

- 仿真参数（时间步长、最大时长等）
- 实体实例（初始位置、属性等）
- 域配置（参数设置）
- 全局设置（阵营关系等）

场景是仿真的起点，决定了仿真的初始条件。

## 场景的组成

### 元数据

场景的基本信息：

- 场景名称
- 版本号
- 描述
- 作者

### 仿真参数

仿真运行时的参数：

- 时间步长
- 最大时长
- 随机种子（可选）

### 实体定义

场景中的实体实例：

- 实体名称
- 实体类型
- 域归属声明
- 组件配置

### 全局配置

全局设置：

- 阵营关系（敌对、中立、友好）
- 环境参数
- 初始条件

## 场景的加载

### 加载流程

1. 解析场景配置文件
2. 创建世界和域
3. 创建实体实例
4. 尝试将实体附加到域
5. 场景就绪

### 加载器的职责

场景加载器负责：

- 解析配置文件格式
- 创建实体和组件
- 处理引用和依赖
- 报告加载错误

## 代理模板

代理模板是实体的预制配置，用于简化场景定义。

### 模板的作用

- 避免重复配置相似的实体
- 提供默认值
- 支持继承和覆盖

### 模板的结构

- 模板名称
- 默认域归属
- 默认组件配置

### 模板的使用

在场景中引用模板：

1. 加载模板定义
2. 基于模板创建实体
3. 覆盖特定属性

## 配置文件格式

配置文件的格式由实现决定，常见选项有 JSON、YAML、TOML 等。

## 场景设计建议

### 实体命名

使用有意义且唯一的名称：

- 包含类型信息：`red_destroyer_01`
- 包含阵营信息：`blue_frigate_alpha`
- 避免使用纯数字

### 组件配置

确保实体具有所声明的域需要的组件：

- 检查域的准入条件
- 配置完整的组件数据
- 避免配置不存在的组件

### 阵营关系

确保阵营关系是对称的：

- 如果 A 对 B 敌对，通常 B 也对 A 敌对
- 避免自相矛盾的关系

### 参数验证

在加载时验证参数的有效性：

- 数值范围检查
- 引用完整性检查
- 依赖关系检查

## 常见问题

### 场景加载失败

常见原因：

- 配置文件格式错误
- 缺少必要的字段
- 引用的模板不存在
- 组件配置不完整

### 实体无法加入域

常见原因：

- 缺少必要的组件
- 组件数据格式错误
- 域的准入规则不满足

### 如何调试场景

建议：

- 使用验证工具检查配置
- 查看加载日志
- 逐步添加实体，定位问题
