# 概览

## 什么是 DUAN

DUAN 是一个通用的仿真体系框架。它的核心设计理念是**权威域架构（Domain-Authoritative Architecture）**——以域（Domain）作为核心计算单元，每个域是其所属领域的唯一权威裁决者，构建可扩展的仿真平台。

与传统仿真系统不同，DUAN 不预设任何具体的仿真逻辑。它提供的是一套**组织仿真计算的框架**，让用户可以根据自己的需求定义和实现具体的仿真行为。

## 核心特点

### 通用性

框架核心不关心具体有哪些域。域的类型、数量、行为都在运行时由用户定义。这意味着：

- 同一套框架可以用于军事仿真、物理仿真、游戏开发等完全不同的领域
- 不需要修改框架代码就能添加新的仿真能力
- 不同的仿真场景可以使用完全不同的域组合

### 可扩展性

可以随时添加新的域类型，而无需修改现有代码。新增域只需要：

1. 定义域的名称和依赖关系
2. 实现域的计算逻辑
3. 注册到域注册表

### 灵活的归属关系

一个实体可以同时属于多个域。实体自己声明要加入哪些域，每个域根据自身规则决定是否接纳。这种设计支持：

- 实体具有多重能力（如同时具有运动能力、探测能力、战斗能力）
- 域可以有不同的准入条件
- 系统自动处理实体与域的关联

### 事件驱动

域之间通过事件系统进行通信，而不是直接调用。这种松耦合设计带来：

- 域之间的依赖关系清晰
- 易于调试和追踪
- 支持回放和分析

## 框架边界

### 框架提供什么

DUAN 框架提供的是**仿真计算的骨架**：

- 实体的管理机制
- 域的注册和调度机制
- 事件的收集和分发机制
- 时间的推进机制
- 仿真循环的组织

### 用户需要提供什么

用户需要提供的是**仿真计算的血肉**：

- 具体的域实现（如空间计算、探测判定、战斗逻辑等）
- 具体的组件定义（如位置、速度、雷达参数等）
- 场景配置（实体、关系、参数等）

## 与其他方案的区别

### 与 ECS（Entity-Component-System）的区别

传统的 ECS 架构中，System 是计算单元，但通常没有明确的"权威"概念——多个 System 可能都在处理同一类判断，导致逻辑分散。在 DUAN 中：

- 域是某个领域的**唯一权威**，同一类判断只有一个域负责
- 所有关于该领域的判定都由对应的域来裁决，不会出现同一问题在多处被回答的情况
- 域之间通过服务接口查询，而不是共享状态

这是 DUAN 与 ECS 最本质的区别：ECS 的 System 是"处理器"，DUAN 的域是"裁决者"。

### 与传统面向对象仿真的区别

传统仿真通常把逻辑封装在实体内部（如"舰船"类包含"探测"方法）。在 DUAN 中：

- 实体只是数据的组合，不包含行为
- 行为由域来实现
- 判定由域来做，不是实体自己判断

这种设计避免了"实体 A 能否看见实体 B"这类判断的归属问题——答案是：由探测域来裁决。
