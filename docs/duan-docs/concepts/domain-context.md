# 域上下文

域上下文（DomainContext）是域与仿真环境交互的唯一入口。本文档描述域上下文的组成、访问权限和使用方式。

## 什么是域上下文

域在执行计算时，不直接持有对仿真组件的引用，而是通过框架在调用计算接口时传入的域上下文来访问仿真环境。

这种设计确保：

- 框架可以在传入上下文时施加访问约束（如只读限制）
- 域的依赖关系清晰，不会隐式持有全局状态
- 便于测试：测试时可以传入 mock 的上下文

## 上下文的组成

### 实体存储（只读）

提供对所有实体及其组件的只读访问。

**用途**：

- 遍历域的实体列表
- 读取实体的组件数据
- 按类型查询实体

**约束**：计算阶段只能读取，不能修改实体数据。

### 域注册表（只读）

提供对其他已注册域的访问，用于域间服务查询。

**用途**：

- 获取其他域的引用，调用其服务接口
- 查询依赖域的计算结果

**约束**：只能查询，不能修改注册表结构。

常见的便捷访问方式（取决于框架实现）：

```
context.domain<SpaceDomain>()   // 类型查找（推荐）
context.domain("space")         // 名称查找（动态场景）
```

### 事件通道（只写）

域发出事件的出口。

**用途**：

- 将计算结果以事件的形式写入通道
- 事件在当前帧的事件处理阶段统一处理

**约束**：只能写入，不能读取已有事件。

### 仿真时钟（只读）

提供当前帧的时间信息。

**字段**：

- **当前仿真时间**：从仿真开始到现在的累计仿真时间
- **帧时间（Delta Time）**：本帧的仿真时间步长

**用途**：

- 运动计算中的时间步长
- 检查相对于仿真时间的条件

**约束**：只能读取，域不能修改时钟状态。

## 使用示例

域在计算接口中使用上下文的典型流程：

```
// 计算接口（伪代码）
function calculate(context):
    // 1. 获取时间信息
    deltaTime = context.clock.deltaTime

    // 2. 遍历本域的实体
    for entity in self.entities:
        // 3. 读取组件数据
        position = entity.get<PositionComponent>()
        movement = entity.get<MovementComponent>()

        // 4. 查询其他域的服务（如有依赖）
        campDomain = context.domain<CampDomain>()

        // 5. 执行计算逻辑
        newPosition = position + movement.velocity * deltaTime

        // 6. 发出事件（不直接修改实体）
        context.events.emit(MoveEvent { entity, newPosition })
```

## 上下文的生命周期

域上下文由框架在每次调用计算接口时创建，计算完成后销毁。域不应该在计算接口外持有上下文的引用。

域的跨帧状态应该存储在域自身的内部状态中，而不是上下文中。

## 设计考量

### 为什么通过上下文而不是直接引用

如果域直接持有对实体存储、事件通道等组件的引用：

- 框架无法在运行时施加访问约束（如只读限制）
- 域的依赖隐式化，难以测试和替换
- 可能绕过框架的约束直接修改状态

通过上下文：

- 访问权限明确（只读/只写标注清晰）
- 依赖显式传入，便于测试
- 框架可以在不同阶段传入不同权限的上下文

### 计算阶段的只读约束

上下文的实体存储在计算阶段是只读的，这是框架的核心约束。即使底层存储支持写入，上下文也只暴露只读接口，从接口层面阻止违规写入。
