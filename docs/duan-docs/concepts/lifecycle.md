# 生命周期

生命周期描述实体从创建到销毁的状态变迁过程。本文档描述生命周期的概念、状态和处理流程。

## 什么是生命周期

生命周期是实体存在期间的状态序列。每个实体都有一个生命周期状态，表示其当前所处的阶段。

**为什么需要生命周期**：

- 区分活跃实体和非活跃实体
- 支持销毁前的过渡阶段
- 便于资源管理和清理

## 生命周期状态

### 初始化中（Initializing）

实体刚创建，正在完成域归属校验。

**特点**：

- 实体已分配标识
- 组件已初始化
- 正在执行域归属校验，尚未加入任何域
- **不参与任何域的计算**

**转移条件**：

- 域归属校验完成（无论各域是否全部通过）→ 进入活跃状态

**关于准入失败**：

准入校验失败不阻止实体进入活跃状态。实体会进入 Active 状态，只是不在拒绝它的域的实体列表中。框架会记录哪些域拒绝了该实体（及拒绝原因），供用户排查配置问题。

### 活跃（Active）

实体正常活动，参与仿真。

**特点**：

- 实体参与域的计算
- 可以发出和接收事件
- 状态可以变化

**转移条件**：

- 收到销毁事件 → 进入销毁中状态

### 销毁中（Destroying）

实体正在销毁，处于过渡阶段。

**进入此状态时，框架自动执行**：

- 调用所有域的脱离接口，实体从各域完全脱离，不再参与计算
- 取消该实体的所有现有定时器

**特点**：

- 实体仍然存在于实体存储中，可被读取
- 不参与任何域的计算
- 过渡期的具体用途由应用层决定（如播放销毁效果、发送通知等）

**转移条件**：

- 过渡期结束（通常由事件处理器在进入此状态时新建的定时器触发）→ 进入已销毁状态

### 已销毁（Destroyed）

实体已销毁，等待清理。

**特点**：

- 实体不再参与计算
- 等待从系统中移除

**转移条件**：

- 清理阶段 → 实体从系统中移除

## 状态转移图

```
[创建] --> 初始化中
初始化中 --> 活跃（域归属校验完成）
活跃 --> 销毁中（收到销毁事件）
销毁中 --> 已销毁（过渡期结束）
已销毁 --> [清理移除]
```

## 生命周期事件

### 销毁事件

触发实体进入销毁流程的事件。

**销毁原因**：

- 被击中
- 超时
- 主动销毁

### 销毁事件的处理

当收到销毁事件时：

1. 将实体状态设为销毁中
2. 设置过渡期定时器
3. 定时器到期后发出已销毁事件
4. 已销毁事件处理时将实体状态设为已销毁

## 清理流程

在仿真循环的清理阶段：

1. 收集所有已销毁状态的实体
2. 从实体存储中移除这些实体

## 生命周期与域的关系

### 域对生命周期的感知

框架保证域只处理活跃状态的实体：

- 初始化中：**不参与计算**（尚未加入域的实体列表）
- 活跃：正常参与计算
- 销毁中：**不参与计算**（进入此状态时脱离接口已被调用，实体已从所有域完全脱离）
- 已销毁：不参与计算
