# 域注册表

域注册表（DomainRegistry）是管理所有域的核心组件。本文档描述域注册表的概念、职责和使用方式。

## 什么是域注册表

域注册表是一个容器，负责：

- 存储所有注册的域
- 支持按类型或名称查找域
- 管理域的执行顺序

域注册表是仿真世界的核心组成部分，所有域都通过域注册表来注册和访问。

## 域注册表的职责

### 域注册

新的域需要注册到域注册表才能参与仿真。注册时需要提供：

- 域的名称（字符串，唯一标识）
- 域的算法实现

注册后，域就可以被其他组件访问和调用。

### 域查找

域注册表支持两种查找方式：

**按类型查找**

通过域的实现类型来查找域。这种方式提供了类型安全，是域之间相互访问的推荐方式。

**按名称查找**

通过域的名称字符串来查找域。这种方式更灵活，适用于动态场景。

### 执行顺序管理

域注册表根据域的依赖关系，自动计算执行顺序。

**拓扑排序**

框架使用拓扑排序算法，根据域声明的依赖关系确定执行顺序：

- 被依赖的域先执行
- 依赖者后执行
- 检测循环依赖

**执行顺序的缓存**

执行顺序在域注册关系不变时是稳定的，域注册表会缓存计算结果，避免重复计算。

### 循环依赖检测

如果域之间存在循环依赖（A 依赖 B，B 依赖 A），域注册表会在计算执行顺序时检测并报错。

## 域之间的协作

域在计算时通过域上下文访问其他域。关于域上下文的完整描述，参见 [域上下文](./domain-context.md)。

### 服务查询

域可以通过上下文查询其他域，获取服务。

例如，探测域可以查询阵营域来判断敌我关系：

```
campDomain = context.domain<CampDomain>()
isEnemy = campDomain.isEnemy(entityA, entityB)
```

### 通用查询

通过类型查找或名称查找均可访问任意已注册的域：

```
context.domain<SpaceDomain>()   // 类型查找（推荐，编译期类型安全）
context.domain("space")         // 名称查找（动态场景）
```

## 设计考量

### 为什么需要域注册表

如果没有域注册表：

- 域之间需要直接引用，耦合紧密
- 执行顺序需要手动管理
- 新增域需要修改多处代码

有了域注册表：

- 域通过注册表访问其他域，松耦合
- 执行顺序自动管理
- 新增域只需注册，不影响现有代码

### 为什么使用类型查找和名称查找两种方式

**类型查找**：

- 编译时类型检查
- 更好的工具支持
- 推荐用于静态已知的域

**名称查找**：

- 更灵活，支持动态场景
- 适用于配置驱动的场景
- 可用于调试和工具

### 执行顺序的确定性

域注册表确保执行顺序是确定的：

- 相同的依赖关系产生相同的执行顺序
- 便于调试和回放
- 避免因顺序不确定导致的难以复现的问题

## 常见问题

### 域注册的时机

域通常在仿真初始化时注册，在仿真运行期间不再变化。

如果需要动态添加域，需要重新计算执行顺序。

### 域查找失败的处理

如果查找一个不存在的域：

- 类型查找返回空值
- 名称查找返回空值

调用方需要处理查找失败的情况。

### 依赖关系变化

域的依赖关系在注册时确定，运行期间不会变化。

如果需要改变依赖关系，需要重新注册域。
