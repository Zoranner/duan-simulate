# 组件

组件（Component）是实体的数据组成单元。本文档描述组件的概念、设计和使用方式。

## 什么是组件

组件是实体的数据片段，用于描述实体的某个方面。例如：

- 位置组件：描述实体的位置和朝向
- 运动组件：描述实体的运动参数
- 雷达组件：描述实体的雷达配置

组件**只包含数据，不包含行为**。组件的使用方式由域来决定。

## 组件的特点

### 数据单元

组件是纯粹的数据容器。它不包含任何计算逻辑或行为方法。

### 可组合性

一个实体可以有多个组件，通过组合不同的组件来表达不同的特性。

### 类型识别

每个组件有一个类型标识，用于区分不同类型的组件。

### 独立性

组件之间没有依赖关系。一个组件的存在不依赖于另一个组件。

## 组件与域的关系

### 域的准入条件

域在接纳实体时，会检查实体是否具有必要的组件。例如：

- 探测域可能要求实体有位置组件和雷达组件
- 运动域可能要求实体有位置组件和运动组件

如果实体缺少必要的组件，域会拒绝接纳。

### 域使用组件

域在计算时会读取实体组件的数据。例如：

- 空间域读取位置组件来计算距离
- 探测域读取雷达组件来获取探测范围
- 战斗域读取武器组件来计算伤害

### 组件不决定行为

同一个组件在不同的域中可能有不同的使用方式。组件本身不决定如何被使用，这是域的职责。

例如，雷达组件只包含雷达的参数（范围、类型等），如何使用这些参数进行探测是探测域的逻辑。

## 组件的设计原则

### 单一职责

每个组件应该只描述实体的一个方面。如果一个组件包含了多个不相关的数据，应该考虑拆分。

### 数据完整

组件应该包含描述该方面所需的完整数据。相关的数据应该放在同一个组件中。

### 避免冗余

组件之间不应该有数据冗余。如果某个数据可以从其他组件计算得出，就不应该存储在组件中。

### 稳定性

组件的结构应该尽量稳定。频繁修改组件结构会影响使用该组件的域。

## 组件容器

实体通过组件容器来管理其组件。组件容器提供以下操作：

### 添加组件

将组件添加到实体中。如果已存在相同类型的组件，会替换旧的组件。

### 获取组件

通过类型获取组件。这是域读取实体数据的主要方式。

### 检查组件

检查实体是否具有某种类型的组件。这常用于域的准入校验。

### 移除组件

从实体中移除组件。这在某些动态场景中可能用到，但较少见。

组件查询的时间复杂度为常数级，确保大规模仿真时的性能。
