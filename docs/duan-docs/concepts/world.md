# 世界

世界（World）是仿真系统的顶层容器，也是整个仿真循环的协调者。

## 组成

世界包含以下核心组件：

- **仿真时钟（TimeClock）**：管理仿真时间、时间比例、暂停恢复
- **域注册表（DomainRegistry）**：管理所有域的注册、查找和执行顺序
- **实体存储（EntityStore）**：管理所有实体的存储、查找和迭代
- **事件通道（EventChannel）**：收集计算阶段的事件，供事件处理阶段分发
- **定时器管理器（TimerManager）**：调度和触发定时器事件

## 职责

世界负责三件事：

- **执行仿真循环**：按阶段顺序驱动每一帧（时间推进 → 域计算 → 定时器检查 → 事件处理 → 清理）
- **管理实体生命周期**：创建实体时将其附加到声明的域，销毁时从所有域移除并清理资源
- **分发事件**：根据事件类型调用对应的事件处理器

## 初始化顺序

```
1. 创建仿真时钟
2. 创建域注册表
3. 创建实体存储
4. 创建事件通道
5. 注册域（含执行顺序解析）
6. 注册事件处理器
7. 加载场景（创建初始实体）
```

域和事件处理器需要在加载场景之前注册完毕，确保实体创建时域和处理器都已就绪。

## 唯一性

一个仿真系统通常只有一个世界实例。多个世界适用于完全独立的并行仿真场景。
