# 时间

时间是仿真的基础维度。本文档描述 DUAN 仿真体系的时间管理概念和机制。

## 时间不是域

时间不是域，而是**基础设施**。所有域都需要访问时间，时间是仿真的元数据。

这意味着：

- 时间由仿真世界统一管理，不由某个域负责
- 所有域通过上下文访问时间，而不是通过域查询
- 时间的推进是仿真循环的一部分，不是域的计算结果

## 仿真时间 vs 真实时间

### 仿真时间

仿真时间是仿真世界中的时间。它：

- 从零开始，随仿真推进而增长
- 与真实时间可能不同步
- 可以加速、减速、暂停

### 真实时间

真实时间是现实世界中的时间。它：

- 由系统时钟提供
- 始终以相同速度流逝
- 用于计算仿真时间的推进量

### 时间比例

时间比例是仿真时间与真实时间的比值：

- 时间比例为 1.0：仿真时间与真实时间同步（实时仿真）
- 时间比例为 2.0：仿真时间是真实时间的两倍（2 倍速）
- 时间比例为 0.5：仿真时间是真实时间的一半（半速）
- 时间比例为 0：仿真时间暂停

## 仿真时钟

仿真时钟是管理仿真时间的组件。

### 时钟的属性

**当前仿真时间**

从仿真开始到现在经过的仿真时间（通常以秒为单位）。

**时间比例**

仿真时间与真实时间的比值。

**运行状态**

时钟是否正在运行。暂停时时钟不会推进。

**步数计数**

已经执行的仿真步数。

## 帧时间

帧时间是每一帧的仿真时间步长。

帧时间的计算：

- 帧时间 = 真实帧时间 × 时间比例
- 如果暂停，帧时间为零

帧时间用于：

- 运动计算：根据帧时间计算位置变化
- 动画播放：根据帧时间推进动画
- 定时器检查：根据仿真时间检查定时器

## 时间在架构中的位置

仿真时钟是仿真世界的一部分，通过域上下文提供给域。

### 域上下文中的时间访问

域通过上下文访问时间信息：

- 当前仿真时间
- 当前帧时间

域不应该直接访问系统时钟，而应该通过上下文获取时间信息。

### 定时器

某些实体需要在特定时间点触发事件。通过定时器机制实现：

- 定时器在指定时间点触发
- 触发时产生定时器事件
- 定时器事件在事件处理阶段被处理

定时器的设计：

- 定时器可以是单次触发或周期触发
- 定时器与实体关联，实体进入销毁中状态时定时器自动取消
- 定时器使用仿真时间，不受时间比例影响（时间比例只影响推进速度）

## 时间的确定性

仿真时间应该是单调递增且确定的。相同的输入（初始条件 + 帧时间序列）应该产生相同的仿真结果。

建议使用固定的帧时间，或者记录帧时间序列用于回放。
