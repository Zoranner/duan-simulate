# 事件

事件（Event）是域之间通信的机制。本文档描述事件的概念、类型和处理流程。

## 什么是事件

事件是域的计算结果的表达方式。当域完成计算后，通过发出事件来通知仿真系统。

**核心特征**：

- 事件是单向的：域发出事件，不等待响应
- 事件是异步的：发出后不会立即处理
- 事件是可追踪的：所有事件都可以被记录和分析

## 事件的作用

### 解耦

域不需要知道谁会处理它的事件，也不需要知道其他域的状态。域只需要发出事件，由仿真系统统一处理。

### 可追溯性

所有事件都可以被记录，用于：

- 调试：追踪状态变化的原因
- 回放：重现仿真过程
- 分析：统计和挖掘仿真数据

### 状态一致性

通过将计算和状态更新分离，确保：

- 计算阶段所有域看到的状态是一致的
- 状态更新在统一的阶段进行
- 避免因执行顺序导致的不一致

## 事件的类型

事件类型完全由用户定义，框架不预设任何事件类型。

常见的事件类型包括：

### 状态变化事件

描述实体状态的变化。

例如：

- 移动事件：实体位置变化
- 属性变化事件：实体属性值变化

### 关系事件

描述实体之间的关系变化。

例如：

- 探测事件：一个实体探测到另一个实体
- 命中事件：一个实体击中另一个实体

### 生命周期事件

描述实体的生命周期变化。

例如：

- 销毁事件：触发实体进入销毁流程
- 已销毁事件：过渡期结束，实体进入已销毁状态

## 事件的内容

每个事件包含：

### 事件类型

标识这是什么类型的事件，决定如何处理。

### 相关实体

事件涉及的实体标识。

### 时间戳

事件发生时的仿真时间。

### 事件数据

事件的具体内容，不同类型的事件有不同的数据结构。

## 事件通道

事件通道是事件的临时容器。计算阶段域将事件写入通道，事件处理阶段事件被逐一取出并处理，每帧结束后通道清空。详见 [仿真循环](../architecture/simulation-loop.md)。

## 事件处理器

事件处理器是与特定事件类型绑定的处理逻辑。域在计算阶段有权直接修改自身管辖实体的组件状态（这是"域是权威"的直接体现）；事件处理器负责处理**跨边界**的状态变更——包括生命周期操作（实体创建/销毁）、对非本域实体的影响，以及来自外部观察者的反馈。

### 处理器的职责

事件处理器可以执行：

- **更新实体状态**：修改组件数据
- **创建新实体**：在仿真中生成新实体
- **销毁实体**：将实体生命周期状态置为销毁中或已销毁
- **产生新事件**：向事件通道写入新事件（新事件在下一帧处理）
- **操作定时器**：创建、取消定时器

### 注册与执行上下文

事件处理器在世界初始化时注册，通过事件类型与处理逻辑绑定。每种事件类型可以注册多个处理器，按注册顺序依次调用。

处理器执行时可以访问：实体存储（读写）、事件通道（只写，新事件下一帧处理）、定时器管理器、仿真时钟（只读）。

### 处理器产生的新事件

事件处理器可以产生新事件，但新事件**在下一帧的事件处理阶段才被处理**，不在本帧处理。这保证单帧内的处理是有界的，不会因事件链式触发导致无限循环。

### 处理顺序

事件按它们被添加到事件通道的顺序处理，域的执行顺序决定了事件的先后。

### 设计建议

- **处理器应保持简单**：只做与该事件直接相关的事。复杂的跨实体计算应在域中完成，处理器只负责把计算结果写入状态
- **事件数据应自包含**：事件携带的数据应包含处理器所需的全部信息，避免处理器反查大量外部数据
- **不能调用域的服务接口**：事件处理阶段域上下文不可用，处理器无法访问域服务。若需要某个判断结果，域计算时应将其包含在事件数据中
