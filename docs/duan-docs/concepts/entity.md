# 实体

实体（Entity）是仿真中的基本对象单元。本文档描述实体的概念、结构和行为。

## 什么是实体

实体是仿真中的一个对象，如舰船、飞机、导弹、编队等。实体是组件的容器，通过组合不同的组件来表达不同的能力。

**关键特征**：

- 实体本身不包含行为逻辑，只包含数据
- 实体的行为由其所属的域来决定
- 实体通过组件组合来表达其特性

## 实体的组成

### 标识

每个实体有一个唯一标识，用于在仿真中区分不同的实体。

标识一旦创建就不会改变，即使实体被销毁，其标识也不会被复用。

### 类型

实体有一个类型字段，用于分类和展示。这是一个字符串值，完全由用户定义。

类型的作用：

- 分类统计：统计各类实体的数量
- 前端展示：选择图标、渲染样式
- 场景加载：匹配代理模板

类型的示例值：

- `"ship"`（舰船）、`"destroyer"`（驱逐舰）、`"frigate"`（护卫舰）
- `"aircraft"`（飞机）、`"helicopter"`（直升机）
- `"missile"`（导弹）
- `"fleet"`（编队）

类型是开放的，用户可以根据需要定义任何类型。

### 域归属

实体声明自己要加入哪些域。这是一个字符串集合，每个字符串对应一个域的名称。

例如，一个驱逐舰可能声明：

- `"space"`（空间域）—— 需要位置管理
- `"movement"`（运动域）—— 需要运动计算
- `"detection"`（探测域）—— 需要探测能力
- `"combat"`（战斗域）—— 需要战斗能力

实体声明域归属后，每个域会根据自身的准入规则决定是否接纳该实体。

### 组件容器

实体包含一个组件容器，用于存储各种组件。组件是实体的数据组成单元。

组件容器支持：

- 添加组件
- 获取组件
- 检查组件是否存在
- 移除组件

组件查询的时间复杂度为常数级，确保大规模仿真时的性能。

### 生命周期状态

实体有一个生命周期状态，表示其当前所处的阶段。详见 [生命周期](./lifecycle.md)。

### 调试日志

实体可以可选地记录调试日志，用于追踪其状态变化。调试日志可以通过配置关闭以节省内存。

## 实体与域的关系

### 自声明归属

实体创建时声明要加入哪些域。这是一种"申请"行为，而不是"强制"行为。

### 域的准入校验

每个域有自己的准入规则。例如：

- 探测域可能要求实体必须有位置组件和雷达组件
- 运动域可能要求实体必须有位置组件和运动参数组件

如果实体不满足某个域的准入规则，该域会拒绝接纳该实体。

### 多域归属

一个实体可以同时属于多个域。这意味着：

- 实体可以具有多种能力
- 不同域可以对同一实体进行不同维度的计算
- 域之间通过事件协调对实体的操作

### 归属的动态性

实体的域归属在创建时确定，但在某些情况下可能发生变化：

- 实体销毁时，会从所有域中脱离
- 域可能因为条件变化而主动脱离某个实体（较少见）
