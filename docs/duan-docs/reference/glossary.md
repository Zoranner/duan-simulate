# 术语表

本文档定义 DUAN 仿真体系中的核心术语。

## 核心概念

### 实体（Entity）

仿真中的基本对象单元。实体是组件的容器，通过组合不同的组件来表达不同的能力。实体本身不包含行为逻辑。

### 组件（Component）

实体的数据组成单元。组件只包含数据，不包含行为。每个组件描述实体的一个方面，如位置、速度、雷达配置等。

### 域（Domain）

仿真中某个领域的权威计算单元。域负责该领域内的所有计算和判定。例如，探测域负责判断一个实体能否探测到另一个实体。

### 域注册表（DomainRegistry）

管理所有域的组件。负责域的注册、查找和执行顺序管理。

### 事件（Event）

域的计算结果的表达方式。域通过发出事件来通知仿真系统，事件在事件处理阶段被统一处理。

### 事件通道（EventChannel）

收集和分发事件的组件。计算阶段域将事件写入事件通道，事件处理阶段事件被逐一处理。

## 架构组件

### 世界（World）

仿真的顶层容器，包含仿真时钟、域注册表、实体存储、事件通道等组件，负责协调仿真循环。

### 仿真时钟（TimeClock）

管理仿真时间的组件。负责时间推进、时间比例管理、暂停恢复等功能。

### 实体存储（EntityStore）

管理所有实体的组件。提供实体的存储、查找、迭代等功能。

### 域上下文（DomainContext）

域访问仿真环境的唯一入口。提供实体存储（只读）、域注册表（只读）、事件通道（只写）、仿真时钟（只读）的访问。框架在调用域的计算接口时传入，域通过上下文与仿真环境交互，而不是直接持有各组件的引用。详见 [域 · 域上下文](../concepts/domain.md#域上下文)。

### 事件处理器（EventHandler）

与特定事件类型绑定的处理逻辑。事件处理阶段框架从事件通道取出事件并调用对应的处理器。事件处理器是仿真中**唯一合法修改实体状态的地方**，在世界初始化时注册。详见 [事件 · 事件处理器](../concepts/event.md#事件处理器)。

### 定时器管理器（TimerManager）

管理所有定时器的组件。负责定时器的调度、检查和触发。

## 设计概念

### 权威域架构（Domain-Authoritative Architecture）

DUAN 的核心设计理念。以域作为核心计算单元，每个域是某个领域的唯一权威，负责该领域的所有计算和判定。与 ECS 中 System 作为"处理器"不同，DUAN 的域是"裁决者"——对领域内的所有问题拥有最终裁决权。

### 主控域模式

当一个判断涉及多个域时，由一个域作为主控者，按需查询其他域的服务，综合判断后发出事件。

### 准入校验

域在接纳实体时进行的校验。域根据自身规则检查实体是否满足条件（如是否具有必要的组件）。

### 域归属

实体与域之间的关联关系。实体通过声明域归属来表达自己要加入哪些域。

### 依赖关系

域之间的关系。一个域可能依赖其他域的信息，依赖关系决定域的执行顺序。

### 拓扑排序

根据依赖关系确定执行顺序的算法。确保被依赖的域先执行。

## 时间概念

### 仿真时间（Simulation Time）

仿真世界中的时间。从零开始，随仿真推进而增长，可以加速、减速或暂停。

### 真实时间（Real Time）

现实世界中的时间。由系统时钟提供，始终以相同速度流逝。

### 时间比例（Time Scale）

仿真时间与真实时间的比值。1.0 表示实时，大于 1.0 表示加速，小于 1.0 表示减速，0 表示暂停。

### 帧时间（Delta Time）

每一帧的仿真时间步长。用于计算位置变化、动画推进等。

## 生命周期

### 初始化中（Initializing）

实体刚创建，正在进行域归属校验的生命周期状态。

### 活跃（Active）

实体正常活动，参与仿真的生命周期状态。

### 销毁中（Destroying）

实体正在销毁，处于过渡阶段的生命周期状态。

### 已销毁（Destroyed）

实体已销毁，等待清理的生命周期状态。
