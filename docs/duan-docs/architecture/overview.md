# 架构概览

本文档描述 DUAN 仿真体系的整体架构，包括职责分层、系统组成和数据流。

## 职责分层

DUAN 采用分层架构，各层职责明确、不越界：

### 概念层

定义仿真体系的核心抽象：实体、组件、域、事件、时间、生命周期。这一层只提供**机制**，不提供任何具体的仿真逻辑。

### 世界层

将各核心组件组合成一个完整可运行的仿真系统。世界是仿真的顶层容器，负责协调仿真循环，管理实体生命周期，分发事件。

### 应用层

用户提供的具体实现：自定义域、自定义组件、事件处理器、场景配置。框架不预设任何应用层内容。

## 系统组成

世界包含以下核心组件，协同完成仿真循环：

```
┌─────────────────────────────────────────────────────────┐
│                        World                            │
│                                                         │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐     │
│  │  TimeClock  │  │ EntityStore │  │EventChannel │     │
│  └─────────────┘  └─────────────┘  └─────────────┘     │
│                                                         │
│  ┌───────────────────────────────────────────────────┐  │
│  │               DomainRegistry                      │  │
│  │  ┌───────┐ ┌───────┐ ┌───────┐ ┌───────┐         │  │
│  │  │Domain │ │Domain │ │Domain │ │Domain │  ...    │  │
│  │  └───────┘ └───────┘ └───────┘ └───────┘         │  │
│  └───────────────────────────────────────────────────┘  │
│                                                         │
│  ┌─────────────┐                                        │
│  │TimerManager │                                        │
│  └─────────────┘                                        │
└─────────────────────────────────────────────────────────┘
```

- **仿真时钟**：管理仿真时间、时间比例、暂停恢复
- **实体存储**：管理所有实体的存储、查找和迭代
- **事件通道**：收集计算阶段的事件，供事件处理阶段分发
- **域注册表**：管理所有域的注册、查找和执行顺序
- **定时器管理器**：调度和触发定时器事件

## 初始化顺序

世界的初始化按以下顺序进行：

```
注册域（含执行顺序解析）
注册事件处理器
加载场景（创建初始实体）
```

域和事件处理器需要在加载场景之前注册完毕，确保实体创建时两者都已就绪。

## 数据流

### 计算阶段

```
域读取实体数据 → 域执行计算 → 域发出事件 → 事件进入事件通道
```

### 事件处理阶段

```
事件通道 → 事件处理器 → 更新实体状态
```

### 整体循环

```
帧开始 → 时间推进 → 域计算（按依赖顺序）→ 定时器检查 → 事件处理 → 清理 → 帧结束
```

## 唯一性

一个仿真系统通常只有一个世界实例。多个世界适用于完全独立的并行仿真场景。
