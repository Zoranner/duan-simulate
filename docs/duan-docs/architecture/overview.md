# 架构概览

本文档描述 DUAN 仿真体系的整体架构。

## 分层设计

DUAN 采用分层架构，从底层到顶层依次为：

### 核心层

核心层提供仿真体系的基础抽象和机制：

- **实体（Entity）**：仿真对象的基本单元
- **组件（Component）**：实体的数据组成
- **组件容器（ComponentBag）**：组件的存储和查询
- **事件通道（EventChannel）**：事件的收集和分发

核心层不包含任何具体的仿真逻辑，只提供机制。

### 域层

域层提供域的抽象和管理：

- **域（Domain）**：权威计算单元的抽象
- **域规则（DomainRules）**：域需要实现的接口
- **域注册表（DomainRegistry）**：域的注册、查找和调度
- **域上下文（DomainContext）**：域访问仿真环境的入口

域层定义了域如何与框架交互，但不包含具体的域实现。

### 世界层

世界层将各个组件组合成一个完整的仿真系统：

- **世界（World）**：仿真世界的容器
- **仿真时钟（TimeClock）**：时间管理
- **实体存储（EntityStore）**：实体管理
- **仿真循环**：每帧的执行流程

世界层协调各组件的工作，但不包含具体的仿真逻辑。

### 应用层

应用层是用户提供的具体实现：

- **具体域**：用户根据仿真需求定义的域
- **具体组件**：用户定义的组件
- **事件处理器**：用户定义的事件处理逻辑
- **场景配置**：用户定义的仿真场景

应用层完全由用户定义，框架不预设任何内容。

## 数据流

### 计算阶段

```
域读取实体数据 → 域执行计算 → 域发出事件 → 事件进入事件通道
```

### 事件处理阶段

```
事件通道 → 事件处理器 → 更新实体状态
```

### 整体流程

```
帧开始 → 时间推进 → 域计算（按顺序）→ 定时器检查 → 事件处理 → 清理 → 帧结束
```
