# 世界

世界（World）是仿真系统的顶层容器。本文档描述世界的概念、组成和职责。

## 什么是世界

世界是仿真的整体环境，包含：

- 所有实体
- 所有域
- 仿真时钟
- 事件通道

世界是仿真的"宇宙"，所有仿真活动都在世界中发生。

## 世界的组成

### 仿真时钟

管理仿真时间。

**职责**：

- 记录当前仿真时间
- 管理时间比例
- 支持暂停和恢复

**访问方式**：通过域上下文只读访问。

### 域注册表

管理所有域。

**职责**：

- 存储注册的域
- 提供域查找功能
- 管理域的执行顺序

**访问方式**：通过域上下文只读访问（用于查询其他域）。

### 实体存储

管理所有实体。

**职责**：

- 存储实体
- 提供实体查找功能
- 支持按类型查询

**访问方式**：通过域上下文只读访问（计算阶段），直接访问（事件处理阶段）。

### 事件通道

收集和分发事件。

**职责**：

- 收集域发出的事件
- 提供事件给事件处理器

**访问方式**：通过域上下文只写访问（计算阶段），直接访问（事件处理阶段）。

### 定时器管理器

管理所有定时器。

**职责**：

- 调度定时器
- 检查到期定时器
- 产生定时器事件

**访问方式**：内部使用，不对外暴露。

## 世界的职责

### 协调仿真循环

世界负责执行仿真循环的各个阶段：

1. 时间推进
2. 域计算
3. 定时器检查
4. 事件处理
5. 清理

### 管理实体生命周期

世界负责实体的创建和销毁：

- 创建实体时，尝试将其附加到声明的域
- 销毁实体时，从所有域中移除并清理资源

### 处理事件

世界负责事件的分发和处理：

- 根据事件类型调用对应的处理器
- 执行事件处理器的逻辑

## 世界与域的关系

### 域的注册

域在世界初始化时注册。注册后：

- 域被添加到域注册表
- 域的依赖关系被解析
- 域被纳入执行顺序

### 域的执行

世界在仿真循环中调用域的计算：

- 按拓扑顺序获取域
- 为每个域创建上下文
- 调用域的计算接口

### 域的访问

域通过上下文访问世界的其他组件：

- 实体存储（只读）
- 域注册表（只读）
- 事件通道（只写）
- 仿真时钟（只读）

## 世界与实体的关系

### 实体的创建

实体通过世界创建：

1. 创建实体对象
2. 初始化组件
3. 声明域归属
4. 尝试附加到各域
5. 添加到实体存储

### 实体的销毁

实体通过事件销毁：

1. 收到销毁事件
2. 进入销毁中状态
3. 销毁动画完成
4. 进入已销毁状态
5. 清理阶段移除

## 世界的初始化

### 初始化顺序

1. 创建仿真时钟
2. 创建域注册表
3. 创建实体存储
4. 创建事件通道
5. 注册所有域
6. 加载场景（创建实体）

### 初始化的职责

世界初始化负责：

- 设置仿真参数（时间比例等）
- 注册所有需要的域
- 加载初始场景

## 设计考量

### 为什么需要世界

如果没有世界这个概念：

- 各组件需要相互引用，耦合紧密
- 仿真循环的协调困难
- 初始化顺序难以管理

有了世界：

- 各组件通过世界协调
- 仿真循环有明确的入口
- 初始化有明确的顺序

### 世界的大小

世界是仿真的顶层容器，应该是唯一的。一个仿真系统通常只有一个世界实例。

### 世界的持久化

世界可以支持持久化：

- 保存当前状态到文件
- 从文件恢复状态

这对于：

- 长时间仿真：保存进度，稍后继续
- 调试：保存问题现场，稍后分析
- 回放：记录状态变化，事后回放

## 常见问题

### 可以有多个世界吗

可以，但通常不需要。多个世界适用于：

- 完全独立的仿真场景
- 并行运行多个仿真

### 世界可以被重置吗

可以。重置世界通常涉及：

- 清空实体存储
- 重置仿真时钟
- 重置域状态

### 世界的线程安全性

世界的线程安全性取决于实现。通常：

- 仿真循环在单线程中执行
- 外部可以并发读取（如果允许）
- 外部写入需要同步
