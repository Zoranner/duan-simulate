# 仿真循环

仿真循环是每帧执行的流程。本文档描述仿真循环的阶段和执行方式。

## 仿真循环概述

仿真循环是仿真的心跳，每帧执行一次。每一帧，仿真世界会：

1. 推进时间
2. 执行域计算
3. 检查定时器
4. 处理事件
5. 清理销毁实体

这些阶段按顺序执行，保证确定性和一致性。

## 阶段详解

### 阶段一：时间推进

根据真实时间流逝和时间比例，推进仿真时间。

**输入**：真实帧时间（自上一帧以来经过的真实时间）

**输出**：仿真帧时间（用于后续计算）

**处理**：

- 如果仿真暂停，帧时间为零
- 如果仿真运行，帧时间 = 真实帧时间 × 时间比例
- 更新仿真时钟

### 阶段二：域计算

按拓扑顺序执行各域的计算。

**执行顺序**：

- 由域注册表根据依赖关系计算
- 被依赖的域先执行
- 依赖者后执行

**每个域的操作**：

1. 获取域上下文
2. 调用域的计算接口
3. 域读取实体数据
4. 域查询其他域（可选）
5. 域发出事件

**约束**：

- 此阶段只能读取实体数据
- 此阶段只能写入事件通道
- 此阶段不能修改实体状态

### 阶段三：定时器检查

检查所有到期的定时器，产生定时器事件。

**处理**：

1. 获取当前仿真时间
2. 检查所有定时器
3. 收集到期的定时器
4. 为每个到期的定时器产生事件
5. 如果是周期定时器，重新调度

**定时器事件的类型**：

- 自毁事件
- 通用事件
- 自定义回调

### 阶段四：事件处理

按顺序处理事件通道中的所有事件。

**处理方式**：

- 串行处理
- 根据事件类型分发到对应的处理器
- 处理器可以修改实体状态

**事件处理的内容**：

- 更新实体状态
- 创建新实体
- 销毁实体
- 触发其他操作

**约束**：

- 此阶段可以修改实体状态
- 事件处理器可以产生新事件，但**新事件在下一帧的事件处理阶段才会被处理**，不在本帧处理

### 阶段五：清理

清理已销毁的实体。

**处理**：

1. 收集所有已销毁状态的实体
2. 从所有域中移除这些实体
3. 取消这些实体的定时器
4. 从实体存储中移除这些实体
5. 更新调试信息（如果启用）

## 阶段间的约束

### 计算阶段只读

计算阶段（阶段二）只能读取实体数据，不能修改。这确保：

- 所有域看到的状态是一致的
- 执行顺序不会影响计算结果
- 便于并行化（未来扩展）

### 事件处理阶段可写

事件处理阶段（阶段四）可以修改实体状态。这是唯一修改状态的阶段。

### 定时器检查在事件处理之前

定时器事件需要参与事件处理，因此定时器检查在事件处理之前。

## 循环的调用

仿真循环由外部调用驱动。调用方负责：

- 计算真实帧时间
- 调用仿真步骤
- 控制调用频率

典型用法：

- 固定帧率：每固定时间间隔调用一次
- 可变帧率：根据真实时间流逝调用
- 单步执行：手动调用单步

## 确定性

### 什么是确定性

确定性是指：相同的输入和初始条件，产生相同的输出和最终状态。

### 为什么重要

- 可复现：便于调试和测试
- 可回放：可以重现仿真过程
- 可验证：可以验证仿真结果

### 如何保证

- 域的执行顺序固定
- 事件的处理顺序固定
- 不依赖不确定的因素（如随机数需要使用固定种子）
